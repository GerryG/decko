Overview

update, and delete.

(The fourth kind of action, read, does not transform cards, and is associated with views, not events).

Whenever you create, update, or delete a card, the card goes through three phases: * validate makes sure all the data is in order * store puts the data in the database * integrate deals with any ramifications of those changes

Events can be defined on each of these stages
